# Transport Mapping-Style Column Guide

This guide describes the columns in the comparison table generated by:

- `codebase/results_analysis/transport_pre_recon_vs_raw_disaggregated.py`

Main output pattern:

- `results/checkpoint_audit/transport_pre_recon_vs_raw_disaggregated_<economy>_<scenario>.csv`

Legacy file names such as `transport_mapping_style_*.csv` are older naming.

## What this table is for

It compares **pre-reconciliation LEAP-side values** against **raw source values** after mapping.

Primary uses:

- Debug stock/share/efficiency mismatches before reconciliation.
- Feed dashboard and series comparison workflows (`leap_series_comparison.py`).

Important: this table is **not** automatically generated by `codebase/transport_workflow.py` in current code. Generate it separately when needed.

## Stock columns

- `pre_effective_stock`
  - LEAP-side effective stock before reconciliation.
  - Formula (typical): `pre_parent_stock * pre_mode_stock_share * pre_device_share / 10000`.
- `pre_effective_stock_scaled`
  - Optional scaled version of `pre_effective_stock` (often same value when multiplier is `1.0`).
- `pre_parent_stock`
  - Parent stock used in effective-stock calculation.
- `pre_mode_stock_share`
  - Stock share (%) at mode/technology parent level.
- `pre_device_share`
  - Device share (%) at fuel-branch level.
- `pre_device_share_sibling_sum`
  - Sum of sibling `Device Share` values in normalized checkpoint state.
- `pre_device_share_sibling_count`
  - Number of siblings used in that sum.
- `pre_norm_device_share_sibling_sum`
  - Sibling share sum from pre-normalization checkpoint.
- `pre_norm_device_share_sibling_count`
  - Sibling count for pre-normalization sum.
- `raw_stocks`
  - Raw source stocks for mapped source tuple (converted to absolute units).
- `stock_pct_diff`
  - `(pre_effective_stock - raw_stocks) / raw_stocks`.
- `stock_pct_diff_scaled`
  - `(pre_effective_stock_scaled - raw_stocks) / raw_stocks`.
- `stock_cause_flag`
  - Heuristic mismatch label for quick triage.

## Mileage / Efficiency / Activity / Intensity columns

- `pre_mileage`, `raw_mileage`, `mileage_pct_diff`
- `pre_efficiency`, `raw_efficiency`, `efficiency_pct_diff`
  - Raw efficiency is converted from `Billion_km_per_pj` to `MJ/100km` before comparison.
- `pre_effective_activity`, `raw_activity`, `activity_pct_diff`
- `pre_intensity`, `raw_intensity`, `intensity_pct_diff`
- `any_metric_over_5pct`
  - `True` when any checked metric has absolute pct difference > 5%.

## Mapping relationship columns

- `leap_branches_per_source_tuple`
- `source_tuples_per_leap_branch`
- `mapping_relationship_type`
  - One of:
    - `one_to_one`
    - `many_leap_to_one_source`
    - `one_leap_to_many_source`
    - `many_to_many`

## Optional reverted-family stock columns

These appear when comparison is run with alt-fuel collapse enabled (`include_alt_fuels_in_fossil_families_stock_view=True`):

- `raw_stocks_reverted`
- `pre_effective_stock_reverted`
- `pre_effective_stock_scaled_reverted`
- `stock_pct_diff_reverted`
- `stock_pct_diff_scaled_reverted`
- `stock_cause_flag_reverted`

Helper/source breakout columns may also appear:

- `source_transport_type`, `source_medium`, `source_vehicle_type`, `source_drive`, `source_fuel`, `reverted_stock_family_fuel`

## Quick interpretation tips

- Large `stock_pct_diff_scaled` with normal mileage/efficiency diffs usually means stock/share allocation issues.
- If fuel-split logic is intentional, use reverted-family columns before concluding stock mismatch.
- `pre_norm_device_share_sibling_sum` far from 100 indicates pre-normalization share-family issues.
- Non-`one_to_one` mapping relationship types need mapping-cardinality review before data-quality conclusions.
